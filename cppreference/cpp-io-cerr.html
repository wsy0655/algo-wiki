std::cerr, std::wcerr

< cpp‎ | io
在标头 <iostream> 定义
extern std::ostream cerr;
 (1)
extern std::wostream wcerr;
 (2)

全局对象 std::cerr 和 std::wcerr 控制到实现定义类型（分别从 std::streambuf 和 std::wstreambuf 派生）的流缓冲区的输出，它与标准 C 错误输出流 stderr 关联。

保证这些对象在构造首个 std::ios_base::Init 类型对象之前或期间得到初始化，而且可以在带有序初始化的静态对象的构造函数和析构函数中使用（只要在定义对象前包含 <iostream>）。

除非发出了 std::ios_base::sync_with_stdio(false)，从多个线程为进行有格式和无格式输出而访问这些对象是安全的。

一旦初始化，那么 (std::cerr.flags() & unitbuf) != 0（std::wcerr 也一样），这表示任何发送给这些流对象的输出都被立即冲洗到 OS（通过 std::basic_ostream::sentry 的析构函数）。

另外，std::cerr.tie() 返回 &std::cout（对 std::wcerr 和 std::wcout 相同），这表示 std::cerr 上的任何输出都会（通过 std::basic_ostream::sentry 的构造函数）首先执行 std::cout.flush()。

注解

名称中的 'c' 指代“字符”（stroustrup.com FAQ）；cerr 表示“字符错误（流）”而 wcerr 表示“宽字符错误（流）”。

示例

通过 std::cerr 输出到 stderr，冲洗 std::cout 上的待处理输出，而通过 std::clog 输出到 stderr 则不会。

#include <chrono>
#include <iostream>
#include <thread>
using namespace std::chrono_literals;
 
void f()
{
    std::cout << "从线程输出...";
    std::this_thread::sleep_for(2s);
    std::cout << "...线程调用 flush()" << std::endl;
}
 
int main()
{
    std::jthread t1{f};
    std::this_thread::sleep_for(1000ms);
    std::clog << "从 main 的这个输出没有绑定到 cout\n";
    std::cerr << "这个输出绑定到了 cout\n";
}

可能的输出：

从 main 的这个输出没有绑定到 cout
从线程输出...这个输出绑定到了 cout
...线程调用 flush()
