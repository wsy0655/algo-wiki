返回指示逆向范围末尾的哨位。

如果 T 是数组类型且 std::remove_all_extents_t<std::remove_reference_t<T>> 不完整，那么调用 ranges::rend 非良构，不要求诊断。

如果实参是左值或者 ranges::enable_borrowed_range<std::remove_cv_t<T>> 是 true，则对 ranges::rend 的调用表达式等价于：

  1. decay-copy(t.rend())(C++23 前)auto(t.rend())(C++23 起)，如果该表达式合法且它的类型实现 std::sentinel_for<decltype(ranges::rbegin(std::declval<T>()))>。
  2. 否则是 decay-copy(rend(t))(C++23 前)auto(rend(t))(C++23 起)，如果 T 是类或枚举类型，而该表达式合法且它的类型实现 std::sentinel_for<decltype(ranges::rbegin(std::declval<T>()))>，其中如同以仅通过进行实参依赖查找来设立 rend 的含义。
  3. 否则是 std::make_reverse_iterator(ranges::begin(std::forward<T>(t)))，如果 ranges::begin(std::forward<T>(t)) 和 ranges::end(std::forward<T>(t)) 都是合法表达式，具有相同类型，且该类型实现 std::bidirectional_iterator。

所有其他情况下，调用 ranges::rend 非良构，这能在 ranges::rend(t) 出现于模板实例化的立即语境时导致代换失败。

定制点对象

名字 ranges::rend 代表一个定制点对象，它是某个字面 semiregular 类类型的 const 函数对象。 细节参见定制点对象 (CustomizationPointObject) 。

注解

如果实参是右值（即 T 是对象类型）且 ranges::enable_borrowed_range<std::remove_cv_t<T>> 是 false，那么调用 ranges::rend 非良构，这也会导致代换失败。

如果 ranges::rend(std::forward<T>(t)) 合法，那么 decltype(ranges::rend(std::forward<T>(t))) 和 decltype(ranges::begin(std::forward<T>(t))) 在所有情况下都实现 std::sentinel_for，同时 T 实现 std::ranges::range。

C++20 标准要求在底层 rend 函数调用返回纯右值时，从实质化的临时对象移动构造其返回值。但所有实现均直接返回纯右值。C++20 后的提案 P0849R8 更正了该要求以匹配实现。

示例

#include <algorithm>
#include <iostream>
#include <ranges>
#include <vector>
 
int main() 
{
    std::vector<int> v = {3, 1, 4};
    namespace ranges = std::ranges;
    if (ranges::find(ranges::rbegin(v), ranges::rend(v), 5) != ranges::rend(v))
        std::cout << "在 vector v 中找到了 5！\n";
 
    int a[] = {5, 10, 15};
    if (ranges::find(ranges::rbegin(a), ranges::rend(a), 5) != ranges::rend(a))
        std::cout << "在数组 a 中找到了 5！\n";
}

输出：

在数组 a 中找到了 5！
