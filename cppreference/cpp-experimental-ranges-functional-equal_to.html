进行比较的函数对象。主模板在 T 类型的 const 左值上调用 operator ==。特化 equal_to<void> 从实参推导函数调用运算符的形参（但非返回类型）。

equal_to 的所有特化均为 Semiregular。

template< class T, class U >

    requires EqualityComparableWith<T, U> ||
             /* std::declval<T>() == std::declval<U>()
                解析成比较指针的内建运算符 */

constexpr bool operator()(T&& t, U&& u) const;
 (2) (仅为 equal_to<void> 特化的成员)
1) 比较 x 与 y。等价于 return ranges::equal_to<>{}(x, y)；
2) 比较 t 与 u，等价于 return std::forward<T>(t) == std::forward<U>(u);，除非表达式解析成调用比较指针的内建 operator ==。

当调用 (1) 或 (2) 会调用比较 P 类型指针的内建运算符时，替而以下列方式确定结果：

  • 若（可能被转换的）第一实参和（可能被转换的）第二实参值，在实现定义的、所有 P 类型指针值上的严格全序中，一者先于另一者，则返回 false。此严格全序与内建运算符 <、>、<= 和 >= 所蕴含的偏序一致。
  • 否则（无一先于另一者）返回 true。

除非从 T 和 U 到 P 的转换序列均保持相等性（见后述），否则行为未定义。

相等性保持

若表达式对给定的相等输入产生相等输出，则它保持相等性。

  • 表达式的输入由其操作数组成。
  • 表达式的输出由其结果和表达式所修改的所有操作数（若存在）组成。

每个要求保持相等性的表达式还进一步要求是稳定的：以相同输入对象对这种表达式进行两次求值必须拥有相等的输出，而无任何对这些输入对象的显式中间修改。

注解

不同于 std::equal_to，ranges::equal_to 要求 == 与 != 均合法（通过 EqualityComparable 和 EqualityComparableWith 制约）。
