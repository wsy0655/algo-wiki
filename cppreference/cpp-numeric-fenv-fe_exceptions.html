FE_DIVBYZERO, FE_INEXACT, FE_INVALID, FE_OVERFLOW, FE_UNDERFLOW, FE_ALL_EXCEPT

< cpp‎ | numeric‎ | fenv


在标头 <cfenv> 定义
#define FE_DIVBYZERO    /* 由实现定义的 2 的幂 */
 (C++11 起)
#define FE_INEXACT      /* 由实现定义的 2 的幂 */
 (C++11 起)
#define FE_INVALID      /* 由实现定义的 2 的幂 */
 (C++11 起)
#define FE_OVERFLOW     /* 由实现定义的 2 的幂 */
 (C++11 起)
#define FE_UNDERFLOW    /* 由实现定义的 2 的幂 */
 (C++11 起)
#define FE_ALL_EXCEPT   FE_DIVBYZERO | FE_INEXACT | \

                        FE_INVALID | FE_OVERFLOW |  \

                        FE_UNDERFLOW
 (C++11 起)

所有这些宏常量（除了 FE_ALL_EXCEPT）都展开成值为 2 的相异次幂整数常量表达式，唯一地鉴别所有受支持浮点数异常。每个宏仅若受支持才得到定义。

宏常量 FE_ALL_EXCEPT 展开成所有其他 FE_* 的逐位或，且始终有定义，若实现不支持浮点数异常，则为零。

常量 定义
FE_DIVBYZERO 出现于之前浮点数运算的极点错误
FE_INEXACT 不准确结果：必须舍入以存储之前浮点数运算的结果
FE_INVALID 出现于之前浮点数运算的定义域错误
FE_OVERFLOW 之前浮点数运算的结果过大而无法表示
FE_UNDERFLOW 之前浮点数运算的结果为有精度损失的非正规值
FE_ALL_EXCEPT 所有受支持浮点数异常的逐位或

实现可于 <cfenv> 定义附加宏常量以鉴别附加浮点数异常。所有这种常量都以 FE_ 后随至少一个大写字母开始。

进一步的细节见 math_errhandling 。

示例

#include <cfenv>
#include <cmath>
#include <iostream>
 
// #pragma STDC FENV_ACCESS ON
 
volatile double zero = 0.0; // 支持 FENV_ACCESS 处不需要 volatile
volatile double one = 1.0;  // 支持 FENV_ACCESS 处不需要 volatile
 
int main()
{
    std::feclearexcept(FE_ALL_EXCEPT);
    std::cout <<  "1.0/0.0 = " << 1.0 / zero << '\n';
    if (std::fetestexcept(FE_DIVBYZERO))
        std::cout << "已报告除以零\n";
    else
        std::cout << "未报告除以零\n";
 
    std::feclearexcept(FE_ALL_EXCEPT);
    std::cout << "1.0/10 = " << one / 10 << '\n';
    if (std::fetestexcept(FE_INEXACT))
        std::cout << "已报告结果不精确\n";
    else
        std::cout << "未报告结果不精确\n";
 
    std::feclearexcept(FE_ALL_EXCEPT);
    std::cout << "sqrt(-1) = " << std::sqrt(-1) << '\n';
    if (std::fetestexcept(FE_INVALID))
        std::cout << "已报告结果无效\n";
    else
        std::cout << "未报告结果无效\n";
}

可能的输出：

1.0/0.0 = inf
已报告除以零
1.0/10 = 0.1
已报告结果不精确
sqrt(-1) = -nan
已报告结果无效
