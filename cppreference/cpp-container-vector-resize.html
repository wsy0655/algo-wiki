std::vector<T,Allocator>::resize

< cpp‎ | container‎ | vector

void resize( size_type count );
 (1) (C++20 起为 constexpr)
void resize( size_type count, const value_type& value );
 (2) (C++20 起为 constexpr)

重设容器大小以容纳 count 个元素，在 count == size() 时不做任何事。

如果当前大小大于 count，那么减小容器到它的前 count 个元素。

如果当前大小小于 count，那么：

1) 追加额外的默认插入的元素。
2) 追加额外的 value 的副本。

参数

count - 容器的大小
value - 用以初始化新元素的值
类型要求
-
为使用重载 (1)， T 必须满足可移动插入 (MoveInsertable) 和可默认插入 (DefaultInsertable) 。
-
为使用重载 (2)， T 必须满足可复制插入 (CopyInsertable) 。

复杂度

与当前大小和 count 间的差成线性。如果容量小于 count 那么就可能有重分配所致的额外复杂度。

异常

如果因为任何原因抛出了异常，那么这些函数无效果（强异常安全保证）。 虽然未明确指定，但当新 vector 所需容量可能超出 max_size() 时会抛出 std::length_error。

重载 (1) 中，如果 T 的移动构造函数不是 noexcept 的且 T 不可复制插入到 *this，那么 vector 将使用会抛出的移动构造函数。如果它抛出，那么豁免此保证且效果未指定。

 (C++11 起)

注解

如果不想要重载 (1) 中的值初始化，例如元素是非类类型且不需要清零，那么可以通过提供定制的 Allocator::construct 避免。
在重设大小到较小值时，vector 的容量决不会减少，因为这会使所有迭代器失效，而仅规定会使指向被擦除元素及其后的迭代器失效。

示例

#include <vector>
#include <iostream>
 
void print(auto rem, const std::vector<int>& c)
{
    for (std::cout << rem; const int el : c)
        std::cout << el << ' ';
    std::cout << '\n';
}
 
int main()
{
    std::vector<int> c = {1, 2, 3};
    print("vector 持有：", c);
 
    c.resize(5);
    print("在增加大小到 5 之后：", c);
 
    c.resize(2);
    print("在减少大小到 2 之后：", c);
 
    c.resize(6, 4);
    print("在增加大小到 6 之后（初始化式 = 4）：", c);
}

输出：

vector 持有：1 2 3
在增加大小到 5 之后：1 2 3 0 0
在减少大小到 2 之后：1 2
在增加大小到 6 之后（初始化式 = 4）：1 2 4 4 4 4

缺陷报告

下列更改行为的缺陷报告追溯地应用于以前出版的 C++ 标准。

缺陷报告 应用于 出版时的行为 正确行为
LWG 679 C++98 resize() 按值传递 value 按 const 引用传递
LWG 1525 C++98 未指定 resize(size()) 的行为 已指定
LWG 2033 C++11 1. 元素通过 erase() 移除[1]
2. T 的类型要求不正确		 1. 通过 pop_back() 移除
2. 已改正
LWG 2066 C++11 重载 (1) 没有重载 (2) 的异常安全保证 已添加
LWG 2160 C++11 LWG 2033 的解决方案要求元素通过 pop_back() 移除[2] 不指定移除元素的方式
  1. ↑ erase() 可以从 vector 的中间移除元素，因此值类型需要可移动赋值 (MoveAssignable) ，这样在移除部分之后的元素可以向前移动以填补空缺。然而 resize() 只能从 vector 的末尾移除元素，这样就不需要可移动赋值 (MoveAssignable) 了。
  2. ↑ 通过 pop_back() 移除元素意味着这些元素需要从后往前移除。
