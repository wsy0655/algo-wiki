注解

从这个分布创建闭区间 \(\small[a, b]\)[a, b] 上的分布较困难。由于舍入误差，以 std::nextafter(b, std::numeric_limits<RealType>::max()) 为第二参数并不总是有效。

多数既存实现都有一个漏洞，它们有时会返回 \(\small b\)b（GCC #63176 LLVM #18767 MSVC STL #1074）。本来以为此为 RealType 是 float 时由 LWG 问题 2524 所导致，但已经发现二者都不是触发这个 BUG 的必要条件。

示例

打印 10 个 1 与 2 间的随机数

#include <iostream>
#include <random>
 
int main()
{
    std::random_device rd;  // 将用于获得随机数引擎的种子
    std::mt19937 gen(rd()); // 以 rd() 播种的标准 mersenne_twister_engine
    std::uniform_real_distribution<> dis(1.0, 2.0);
    for (int n = 0; n < 10; ++n)
        // 用 dis 变换 gen 生成的随机 unsigned int 为 [1, 2) 中的 double
        // 每次调用 dis(gen) 都生成新的随机 double
        std::cout << dis(gen) << ' ';
    std::cout << '\n';
}
