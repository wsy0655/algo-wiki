 (2) (C++17 起)

复制符号链接到另一位置。

1) 等效于调用 f(read_symlink(from), to)，其中 f 是 create_symlink 或 create_directory_symlink，依赖于 from 解析为文件或目录。
2) 等效于调用 f(read_symlink(from, ec), to, ec)，其中 f 是 create_symlink 或 create_directory_symlink，依赖于 from 解析为文件或目录。

参数

from - 要复制的符号链接的路径
to - 新符号链接的目标路径
ec - 不抛出重载中报告错误的输出形参

返回值

（无）

异常

若内存分配失败，则任何不标记为 noexcept 的重载可能抛出 std::bad_alloc 。

1) 抛出 std::filesystem::filesystem_error，构造时以 from 为第一路径实参，以 to 为第二路径实参，并以OS 错误码为错误码实参。
2) 若 OS API 调用失败，则设置 std::error_code& 为 OS API 错误码，而未发生错误时则执行 ec.clear()。
