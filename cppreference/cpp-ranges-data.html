返回指向连续范围首元素的指针。

如果 T 是数组类型且 std::remove_all_extents_t<std::remove_reference_t<T>> 不完整，那么 ranges::data(std::forward<T>(t)) 非良构，不要求诊断。

如果实参是左值或 ranges::enable_borrowed_range<std::remove_cv_t<T>> 是 true，那么对 ranges::data 的调用表达式等价于：

  1. decay-copy(t.data())(C++23 前)auto(t.data())(C++23 起)，如果表达式合法且它的类型是指向对象的指针类型。
  2. 否则，如果表达式 ranges::begin(std::forward<T>(t)) 合法且它的类型实现 std::contiguous_iterator，则是 std::to_address(ranges::begin(std::forward<T>(t)))。

所有其他情况下，对 ranges::data 的调用非良构，这能在 ranges::data(e) 出现于模板实例化的立即语境时导致代换失败。

定制点对象

名字 ranges::data 代表一个定制点对象，它是某个字面 semiregular 类类型的 const 函数对象。 细节参见定制点对象 (CustomizationPointObject) 。

注解

如果实参是右值（即 T 是对象类型）且 ranges::enable_borrowed_range<std::remove_cv_t<T>> 是 false，那么调用 ranges::data 非良构，这也会导致代换失败。

如果 ranges::data(e) 对表达式 e 合法，那么它返回指向对象的指针。

C++20 标准要求在底层 data 函数调用返回纯右值时，从实质化的临时对象移动构造其返回值。但所有实现均直接返回纯右值。C++20 后的提案 P0849R8 更正了该要求以匹配实现。

示例

#include <cstring>
#include <iostream>
#include <ranges>
#include <string>
 
int main()
{
    std::string s {"Hello world!\n"};
 
    char a[20]; // C 风格字符串的存储
    std::strcpy(a, std::ranges::data(s));
    // [data(s), data(s) + size(s)] 保证是 NTBS（空中止字节字符串）
 
    std::cout << a;
}

输出：

Hello world!
