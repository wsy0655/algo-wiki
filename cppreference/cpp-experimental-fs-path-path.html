构造新 path 对象。

1) 构造空路径。
2) 复制构造函数。构造 p 的副本。
3) 移动构造函数。构造 p 的副本，p 遗留为有效单未指明的状态。
4,5) 从 source (4) 所提供的字符序列（它是指针，指向空终止字符/宽字符序列的输入迭代器，或者 std::basic_string）所提供的字符序列，或以一对输入迭代器 [first, last) (5) 表示的字符序列，构造路径。允许四种字符类型 char、char16_t、char32_t、wchar_t 中的任何一种，而转换为原生字符集的方法依赖于 source 所用的字符类型。
  • 如果源字符类型为 char，则假定源编码为原生编码（因而在 POSIX 系统上不会发生转换）。
  • 如果源字符类型为 char16_t，则从 UTF-16 转换为所使用的原生文件系统编码。
  • 如果源字符类型为 char32_t，则从 UTF-32 转换为所使用的原生文件系统编码。
  • 如果源字符类型为 wchar_t，则假定输入为原生宽编码（因而在 Windows 上不会发生转换）。
6,7) 从 source (6) 所提供的字符序列（它是指针，指向空终止字符序列的输入迭代器，或者 std::string）所提供的字符序列，或以一对输入迭代器 [first, last) (7) 表示的字符序列，构造路径。所允许的唯一字符类型为 char。使用 loc 来实施字符编码转换。若 value_type 为 wchar_t，则使用 loc 的 std::codecvt<wchar_t, char, std::mbstate_t> 刻面将其转换为宽字符。否则，首先使用 std::codecvt<wchar_t, char, std::mbstate_t> 刻面将其转换为宽字符，然后使用 loc 的 std::codecvt<wchar_t, value_type> 刻面将其转换为文件系统原生字符类型。

参数

p - 要复制的路径
source - std::basic_string，指向空终止字符串的指针，或具有字符值类型并指向空终止字符序列的输入迭代器（对于重载 (6) 其字符类型必须为 char）
first, last - 一对 老式输入迭代器 (LegacyInputIterator) ，指定 UTF-8 编码的字符序列
loc - 定义所用编码转换的本地设置
类型要求
-
InputIt 必须满足老式输入迭代器 (LegacyInputIterator) 。
-
为使用重载 (5)，InputIt 的值类型必须为四种字符类型 char、wchar_t、char16_t 和 char32_t 之一。
-
为使用重载 (7)，InputIt 的值类型必须为 char。

1,2) （无）

注解

对于来自 Unicode 字符串的可移植路径名，见 u8path。

示例

#include <experimental/filesystem>
#include <iostream>
namespace fs = std::experimental::filesystem;
 
int main()
{
    fs::path p1 = "/usr/lib/sendmail.cf"; // 可移植格式
    fs::path p2 = "C:\\users\\abcdef\\AppData\\Local\\Temp\\"; // 原生格式
    fs::path p3 = L"D:/猫.txt"; // 宽字符串
 
    std::cout << "p1 = " << p1 << '\n'
              << "p2 = " << p2 << '\n'
              << "p3 = " << p3 << '\n';
}

输出：

p1 = "/usr/lib/sendmail.cf"
p2 = "C:\users\abcdef\AppData\Local\Temp\"
p3 = "D:/猫.txt"
