std::is_literal_type

< cpp‎ | types
在标头 <type_traits> 定义
template< class T >
struct is_literal_type;
 (C++11 起)
(C++17 弃用)
(C++20 移除)

std::is_literal_type 是一元类型特征 (UnaryTypeTrait) 。

（由于对泛型编程而言十分鸡肋，这项类型特性已经弃用[1]并移除[2]。）

如果 T 满足字面类型 (LiteralType) 的所有要求，那么提供的成员常量 value 等于 true。对于其它任何类型，value 等于 false。

如果 std::remove_all_extents_t<T> 是除了（可有 cv 限定的）void 以外的不完整类型，那么行为未定义。

如果程序添加了 std::is_literal_type 或 std::is_literal_type_v 的特化，那么行为未定义。

成员类型

类型 定义
value_type bool

注解

只有字面类型能作为 constexpr 函数的形参或返回类型。只有字面类能拥有 constexpr 成员函数。

示例

#include <type_traits>
 
struct A { int m; };
static_assert(std::is_literal_type_v<A> == true);
 
struct B { virtual ~B(); };
static_assert(std::is_literal_type_v<B> == false);
 
int main() {}
