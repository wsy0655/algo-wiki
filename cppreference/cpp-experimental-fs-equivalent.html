std::experimental::filesystem::equivalent

< cpp‎ | experimental‎ | fs
在标头 <experimental/filesystem> 定义
bool equivalent( const path& p1, const path& p2 );
bool equivalent( const path& p1, const path& p2, error_code& ec );
 (1) (文件系统 TS)

检查路径 p1 和 p2 是否指代同一个文件或目录，并由如同以 status 确定具有相同的文件状态（跟随符号链接）。

若 p1 或 p2 不存在或它们的文件类型并非文件、目录或符号链接（由 is_other 确定），则报告错误。

无抛出重载当发生错误时返回 false。

参数

p1, p2 - 要检查等价性的路径
ec - 用于无抛出重载中报告错误的输出形参

返回值

如果 p1 和 p2 指代相同文件或目录且它们的文件状态相同，则为 true，否则为 false。

异常

不接受 error_code& 形参的重载，在发生底层 OS API 错误时抛出 filesystem_error，它以 p1 为第一实参，以 p2 为第二实参，并以 OS 错误码为错误码实参构造。如果内存分配失败，则可抛出 std::bad_alloc。接受 error_code& 形参的重载，当 OS API 调用失败时将之设置为 OS API 错误码，而未发生错误时执行 ec.clear()。此重载具有
noexcept 规定：  
noexcept
  

注解

对于两个路径，如果如同以 POSIX stat 获得的 POSIX stat 结构体的 st_dev 和 st_ino 相等（意为两个文件位于相同设备并处于相同位置），则认为它们解析为相同的文件系统实体。

特别是，同一个文件或目录的所有硬链接均等价，且同一文件系统上的符号链接及其目标等价。

示例

#include <cstdint>
#include <experimental/filesystem>
#include <iostream>
namespace fs = std::experimental::filesystem;
 
int main()
{
    // 硬链接等价
    fs::path p1 = ".";
    fs::path p2 = fs::current_path();
    if (fs::equivalent(p1, p2))
        std::cout << p1 << " 等价于 " << p2 << '\n';
 
    // 符号链接等价
    fs::path p3 = "/lib/libc.so.6";
    fs::path p4 = p3.parent_path() / fs::read_symlink(p3);
    if (fs::equivalent(p3, p4))
        std::cout << p3 << " 等价于 " << p4 << '\n';
}

可能的输出：

"." 等价于 "/var/tmp/test"
"/lib/libc.so.6" 等价于 "/lib/libc-2.12.so"
