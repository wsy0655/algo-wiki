返回范围 [first, last) 中首个满足特定准则的元素：

1) find 搜索投射后的值等于 value 的元素（即 value == ranges::invoke(proj, *i)）。
3) find_if 搜索谓词 q 为其投射后的值返回 true 的元素（即 ranges::invoke(pred, ranges::invoke(proj, *i)) 为 true）。
5) find_if_not 搜索谓词 q 为其投射后的值返回 false 的元素（即 ranges::invoke(pred, ranges::invoke(proj, *i)) 为 false）。
2,4,6) 与 (1,3,5) 相同，但使用 r 为源范围，如同以 ranges::begin(r) 为 first 并以 ranges::end(r) 为 last。

尽管声明描述如上，算法声明的模板形参的实际数量和顺序是未指定的。从而若在调用算法时使用显式模板实参，则程序很可能不可移植。

参数

first, last - 要检验的元素范围
r - 要检验的元素范围
value - 与投射后元素比较的值
pred - 运用于投射后元素的谓词
proj - 运用于元素的投射

返回值

指向首个满足条件的元素的迭代器。如果未找到这种元素，则返回与 last 比较相等的迭代器。

复杂度

至多 last - first 次运用谓词和投射。

可能的实现

版本一 
template<InputIterator I, Sentinel<I> S, class T, class Proj = ranges::identity>
    requires IndirectRelation<ranges::equal_to<>, projected<I, Proj>, const T*>
I find(I first, S last, const T& value, Proj proj = Proj{})
{
    for (; first != last; ++first)
        if (ranges::invoke(proj, *first) == value)
            break;
    return first;
}
版本二 
template<InputIterator I, Sentinel<I> S, class Proj = ranges::identity,
         IndirectUnaryPredicate<projected<I, Proj>> Pred>
I find_if(I first, S last, Pred pred, Proj proj = Proj{})
{
    for (; first != last; ++first)
        if (ranges::invoke(pred, ranges::invoke(proj, *first)))
            break;
    return first;
}
版本三 
template<InputIterator I, Sentinel<I> S, class Proj = ranges::identity,
         IndirectUnaryPredicate<projected<I, Proj>> Pred>
I find_if_not(I first, S last, Pred pred, Proj proj = Proj{})
{
    for (; first != last; ++first)
        if (!ranges::invoke(pred, ranges::invoke(proj, *first)))
            break;
    return first;
}
