std::atomic<T>::is_lock_free

< cpp‎ | atomic‎ | atomic
bool is_lock_free() const noexcept;
 (1) (C++11 起)
bool is_lock_free() const volatile noexcept;
 (2) (C++11 起)

检查此类型所有对象上的原子操作是否免锁。

参数

（无）

返回值

在此类型所有对象上的原子操作免锁的情况下返回 true，否则返回 false。

注解

std::atomic_flag 以外的所有原子类型可用互斥体或其他锁定操作实现，而不一定用免锁的原子 CPU 指令。也允许原子类型有时 ﻿免锁，例如给定架构上仅对齐的内存访问是天然原子的，此时同类型的未对齐对象就必须用锁。

C++ 标准推荐（但不要求）免锁操作都是免地址的，即适用于使用共享内存的进程间交流。

示例

#include <atomic>
#include <iostream>
#include <utility>
 
struct A { int a[100]; };
struct B { int x, y; };
 
int main()
{
    std::cout << std::boolalpha
              << "std::atomic<A> 是否免锁？"
              << std::atomic<A>{}.is_lock_free() << '\n'
              << "std::atomic<B> 是否免锁？"
              << std::atomic<B>{}.is_lock_free() << '\n';
}

可能的输出：

std::atomic<A> 是否免锁？false
std::atomic<B> 是否免锁？true
