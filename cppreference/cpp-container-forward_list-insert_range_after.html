std::forward_list<T,Allocator>::insert_range_after

< cpp‎ | container‎ | forward list
template< container-compatible-range<T> R >
iterator insert_range_after( const_iterator pos, R&& rg );
 (C++23 起)

以非逆序的顺序，将 rg 中各元素的副本插入到 pos 之前。范围 rg 中的每个迭代器均被恰好解引用一次。

pos 必须为范围 [begin(), end()) 内的任何可解引用迭代器，或为 before_begin() 迭代器（因此 end() 对于 pos 不是有效的实参）。

不使任何迭代器或引用失效。

若 rg 与容器重叠则其行为未定义。

参数

pos - 将于其后插入内容的迭代器
rg - 容器兼容范围，即元素可转换为 T 的 input_range
类型要求
-
T 必须为可从 *ranges::begin(rg) 向 forward_list 可就位构造 (EmplaceConstructible) 。否则，其行为未定义。

返回值

指向最后插入到 forward_list 的元素副本，或当 rg 为空时指向 pos 的迭代器。

复杂度

对于 rg 的大小为线性。

注解

功能特性测试宏 值 标准 功能特性
__cpp_lib_containers_ranges 202202L (C++23) 范围感知构造和插入

示例

#include <algorithm>
#include <cassert>
#include <forward_list>
#include <iterator>
#include <vector>
 
int main()
{
    auto container = std::forward_list{1, 2, 3, 4};
    auto pos = std::next(container.cbegin());
    assert(*pos == 2);
    const auto rg = std::vector{-1, -2, -3};
 
#ifdef __cpp_lib_containers_ranges
    container.insert_range_after(pos, rg);
#else
    container.insert_after(pos, rg.cbegin(), rg.cend());
#endif
 
    assert(std::ranges::equal(container, std::vector{1, 2, -1, -2, -3, 3, 4}));
}
