n 次推进迭代器 i，或直至抵达 bound，先到为止。

1) 若 I 实现 RandomAccessIterator，则等价于 i += n。否则自增（或若 n 为负则自减）i {{|n}} 次。若 n 为负且 I 不满足 BidirectionalIterator 则行为未定义。
2) 若满足 Assignable<I&, S>，则等价于 {{{1}}}。
否则，若 [i, bound) 不代表范围，则行为未定义。
否则，若 SizedSentinel<S, I> 得到满足，则等价于 ranges::advance(i, bound - i)。
否则，自增 i 直至 i == bound。
3) 若 SizedSentinel<S, I> 得到满足，则若 |n| >= |bound - i| 则等价于 ranges::advance(i, bound)，否则等价于 ranges::advance(i, n)。
否则自增（或若 n 为负则自减）i n 次或直至 i == bound，先到为止。
若 n > 0，则 [i, bound) 应当代表一个范围；若 n == 0，则 [i, bound) 或 [bound, i) 应当代表一个范围；若 n < 0，则 [bound, i) 应当代表一个范围，且 I 与 S 应当是同一类型，且 I 必须满足 BidirectionalIterator。否则，行为未定义。

定制点对象

名称 ranges::advance 代表一个定制点对象，它是字面 Semiregular 类类型的cpp/named req/FunctionObject（为说明目的以 AdvanceT 表示）。AdvanceT 的所有实例均相等。从而能自由地复制 ranges::advance，且能交替使用其副本。

给定类型集合 Args...，若 std::declval<Args>()... 满足上面针对 ranges::advance 实参的要求，则 AdvanceT 将满足 ranges::Invocable<const AdvanceT, Args...>。否则，没有 AdvanceT 的函数调用运算符参与重载决议。

在每个定义 ranges::advance 的翻译单元中，它都指代定制点对象的同一实例。（这表示能自由地在类似 inline 函数和函数模板的东西中使用它，而不违反单一定义规则。）

返回值

1,2) （无）
3) 由于抵达 bound 而不进行的自增/自减次数。换言之为 n - M，其中 M 是从 i 的开始位置到结束位置的距离，而若结束位置在开始位置之前则为负。
