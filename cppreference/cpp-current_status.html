最近的里程碑：C++23 已经发布，C++26 正在紧锣密鼓进行中。

C++23 已经成功发布，现在我们正全力以赴地投入 C++26 的工作中。

自 2012 年以来，C++ 委员会已经过渡到了一种“解耦”的模式，其中主要的工作部分可以独立于 C++ 标准本身而进行，并以“功能分支”的技术规范（TS）的形式交付。各个供应商可以选择实施这些 TS，而社区则可以通过使用每个功能的 std::experimental 版本来积累经验。这使得我们可以根据实践经验对每个功能的设计进行学习和调整，然后再将其合并到“主干”C++ 标准之前，以确定最终规范。与此同时，标准的交付可以更加定期，功能批次更小且更可预测。这种方法还有助于C++编译器更紧密地跟踪 C++ 标准，以更一致的顺序添加实验性和草稿最终的 C++ 功能。当前计划详见论文 P1000。

您还可以访问 open-std.org，以获取最新的 C++ 标准委员会文件。通过阅读这些提案，您可以追踪C++的发展趋势，了解一个出色的想法是如何成为标准的。然而，需要注意的是，这些文件不是，也不应被视为标准文件。
