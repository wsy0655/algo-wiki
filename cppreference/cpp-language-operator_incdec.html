自增/自减运算符

< cpp‎ | language


自增/自减运算符增加或减小对象的值。

运算符名 语法 可重载 原型示例（对于类 class T）
类内定义 类外定义
前自增 ++a 是 T& T::operator++(); T& operator++(T& a);
前自减 --a 是 T& T::operator--(); T& operator--(T& a);

注解
  • 内建运算符的前置版本返回引用 ﻿而后置版本返回值，典型的用户定义重载也遵循该模式，从而使用户定义运算符可以与内建版本相同的方式使用。然而，用户定义重载中，能以任何类型为返回类型（包括 void）。
  • int 形参是用于区别运算符的前置和后置版本的空设形参。调用用户定义的运算符时，该形参中传递的值始终为零，但通过函数调用记法调用这个运算符时可以更改它（例如 a.operator++(2) 或 operator++(a, 2)）。

前置运算符

前置自增与自减表达式的形式为

++ 表达式
-- 表达式
1) 前置自增（前自增）
2) 前置自减（前自减）

内建的前置运算符

1) 表达式 ++x 等价于 x += 1，但有以下例外：
  • 如果表达式 ﻿的类型是（可有 volatile 限定的）bool，那么会将表达式 ﻿设为 true。此类自增已弃用。
 (C++17 前)
  • 如果表达式 ﻿的类型是（可有 cv 限定的）bool，那么程序非良构。
 (C++17 起)
  • 如果表达式 ﻿的类型有 volatile 限定，那么此类自增已弃用。
 (C++20 起)
2) 表达式 --x 等价于 x -= 1，但有以下例外：
  • 如果表达式 ﻿的类型是（可有 cv 限定的）bool，那么程序非良构。
  • 如果表达式 ﻿的类型有 volatile 限定，那么此类自减已弃用。
 (C++20 起)

重载

在针对用户定义运算符的重载决议中，对于每个 bool 以外的可选地有 volatile 限定的算术类型 A，和对于每个可选地有 volatile 限定的指向可选地有 cv 限定的对象类型的指针 P，下列函数签名参与重载决议：

A& operator++(A&)
bool& operator++(bool&)
 (弃用)(C++17 前)
P& operator++(P&)
A& operator--(A&)
P& operator--(P&)

后置运算符

后置自增与自减表达式拥有形式

表达式 ++
表达式 --
1) 后置自增（后自增）
2) 后置自减（后自减）

内建的后置运算符

后置自增和自减的结果是对表达式 ﻿（在修改前）应用左值到右值转换得到的值。结果类型是表达式 ﻿的类型的无 cv 限定版本。

如果表达式 ﻿不是具有（可有 cv 限定的）bool 以外的(C++17 起)整数类型或指向完整对象类型的指针类型，那么程序非良构。

如果表达式 ﻿的类型有 volatile 限定，那么此类自增和自减已弃用。

 (C++20 起)
1) 表达式 ﻿的值如同以将它作为前置 ++ 运算符的方式被修改。
2) 表达式 ﻿的值如同以将它作为前置 -- 运算符的方式被修改。

后置自增和自减的值计算按顺序早于对表达式 ﻿的修改。对于顺序不确定的函数调用，后置自增和自减操作会被视为单次求值。

重载

在针对用户定义运算符的重载决议中，对于每个 bool 以外的可选地有 volatile 限定的算术类型 A，和对于每个可选地有 volatile 限定的指向可选地有 cv 限定的对象类型的指针 P，下列函数签名参与重载决议：

A operator++(A&, int)
bool operator++(bool&, int)
 (弃用)(C++17 前)
P operator++(P&, int)
A operator--(A&, int)
P operator--(P&, int)

示例

#include <iostream>
 
int main()
{
    int n1 = 1;
    int n2 = ++n1;
    int n3 = ++ ++n1;
    int n4 = n1++;
//  int n5 = n1++ ++;   // 错误
//  int n6 = n1 + ++n1; // 未定义行为
    std::cout << "n1 = " << n1 << '\n'
              << "n2 = " << n2 << '\n'
              << "n3 = " << n3 << '\n'
              << "n4 = " << n4 << '\n';
}

输出：

n1 = 5
n2 = 2
n3 = 4
n4 = 4

注解

因为涉及到了副作用，所以必须谨慎使用内建的自增和自减运算符，以避免违背求值顺序规则所致的未定义行为。

因为后自增和后自减期间会构造对象的临时副本，所以在不使用返回值的语境中前自增 ﻿或前自减 ﻿运算符通常更有效率。

标准库

缺陷报告

下列更改行为的缺陷报告追溯地应用于以前出版的 C++ 标准。

缺陷报告 应用于 出版时的行为 正确行为
CWG 2855 C++98 内建前自增和前自减会应用一般算术转换，但后自增和后自减不会[1] 也会应用
CWG 2901 C++98 内建后自增和后自减不会应用左值到右值转换 会应用
  1. ↑ 前置形式 ++x 等价于 x += 1，而后者适用一般算术转换（即在 decltype(x) 与 int 之间产生公共类型）。然而后置形式 x++ 的效果只是简单地“对 x 加一”，因为不存在二元运算符，所以也不会有一般算术转换。
