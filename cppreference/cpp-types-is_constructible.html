std::is_constructible, std::is_trivially_constructible, std::is_nothrow_constructible

< cpp‎ | types
在标头 <type_traits> 定义
template< class T, class... Args >
struct is_constructible;
 (1) (C++11 起)
template< class T, class... Args >
struct is_trivially_constructible;
 (2) (C++11 起)
template< class T, class... Args >
struct is_nothrow_constructible;
 (3) (C++11 起)
1) 如果 T 是对象或引用类型，且变量定义 T obj(std::declval<Args>()...); 良构，那么提供的成员常量 value 等于 true。对于其它所有类型，value 等于 false。
对于这项检查而言，决不会把变量定义判读为函数声明，且不认为 std::declval 是 ODR 式使用。如同从与 T 和 Args 中的任何类型无关的语境进行访问检查。仅考虑变量定义的立即语境的合法性。
2) 同 (1)，但变量定义不进行任何非平凡操作。对于这项检查而言，对 std::declval 的调用被认为是平凡的。
3) 同 (1)，但变量定义为 noexcept。

如果 T 或形参包 Args 中的任何类型不是完整类型，（可有 cv 限定的）void 或未知边界数组，那么行为未定义。

如果上述模板的实例化直接或间接地依赖于不完整类型，并且假设在该类型完整的情况下实例化就会产生不同的结果，那么行为未定义。

如果程序添加了此页面上描述的任何模板的特化，那么行为未定义。

注解

多数实现中，is_nothrow_constructible 亦检查析构函数是否抛出，因为它相当于 noexcept(T(arg))。同样的情况适用于 is_trivially_constructible，在这些实现中亦要求析构函数为平凡的：GCC 漏洞 51452 LWG 问题 2116。

示例

#include <iostream>
#include <type_traits>
 
class Foo
{
    int v1;
    double v2;
public:
    Foo(int n) : v1(n), v2() {}
    Foo(int n, double f) noexcept : v1(n), v2(f) {}
};
 
int main()
{
    auto is = [](bool o) { return (o ? "\t" "" : "\t" "不"); };
    std::cout << "Foo ...\n"
              << is(std::is_trivially_constructible_v<Foo, const Foo&>)
              << "可从 const Foo& 平凡构造\n"
              << is(std::is_trivially_constructible_v<Foo, int>)
              << "可从 int 平凡构造\n"
              << is(std::is_constructible_v<Foo, int>)
              << "可从 int 构造\n"
              << is(std::is_nothrow_constructible_v<Foo, int>)
              << "可从 int 无抛出构造\n"
              << is(std::is_nothrow_constructible_v<Foo, int, double>)
              << "可从 int 和 double 无抛出构造\n";
}

输出：

Foo ...
        可从 const Foo& 平凡构造
        不可从 int 平凡构造
        可从 int 构造
        不可从 int 无抛出构造
        可从 int 和 double 无抛出构造
