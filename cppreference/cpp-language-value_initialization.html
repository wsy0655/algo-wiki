值初始化

< cpp‎ | language


这是在以空初始化器构造对象时进行的初始化。

语法

T () (1)
new T () (2)
类::类(...) : 成员() { ... } (3)
T 对象 {}; (4) (C++11 起)
T {} (5) (C++11 起)
new T {} (6) (C++11 起)
类::类(...) : 成员{} { ... } (7) (C++11 起)

解释

以下场合进行值初始化：

1,5) 当以空的括号或花括号(C++11 起)对组成的初始化器创建无名临时对象时；
2,6) 当 new 表达式以空的括号或花括号(C++11 起)对组成的初始化器创建具有动态存储期的对象时；
3,7) 当使用以空的括号或花括号(C++11 起)对组成的成员初始化器初始化非静态数据成员或基类时；
4) 当以由空花括号对组成的初始化器声明具名对象（自动、静态或线程局部）时。

所有情况下，如果使用空花括号对 {} 且 T 是聚合类型，那么就会进行聚合初始化而非值初始化。

如果 T 是没有默认构造函数但带有接受 std::initializer_list 的构造函数的类类型，那么就会进行列表初始化。

 (C++11 起)

值初始化的效果是：

  • 如果 T 是（可有 cv 限定的）类类型：
  • 如果 T 的默认初始化选择了一个构造函数，并且该构造函数不由用户声明(C++11 前)由用户提供(C++11 起)，那么对象会首先被零初始化。
  • 任何情况下，对象都被默认初始化。
  • 否则，如果 T 是数组类型，那么值初始化数组的每个元素。
  • 否则，零初始化对象。

注解

语法 T object(); 不初始化对象；它声明一个不接受参数并返回 T 的函数。在 C++11 前值初始化一个具名变量的方式是 T object = T();，它值初始化一个临时量然后复制初始化该对象：多数编译器在此情况下能优化掉复制。

在 C++03（引入了值初始化）之前的 C++98 中，表达式 new T() 被归类为默认初始化并指定进行零初始化。

引用不能被值初始化。

如函数风格转换中所述，T 指名数组类型时禁止语法 T() (1)，但允许 T{} (5)。

所有标准容器（std::vector、std::list 等）在以单个 size_type 实参进行构造或由对 resize() 的调用而增长时值初始化它们的各个元素，除非它们的分配器定制了 construct 的行为。

示例

#include <cassert>
#include <iostream>
#include <string>
#include <vector>
 
struct T1
{
    int mem1;
    std::string mem2;
    virtual void foo() {} // 确保 T1 不是聚合体
}; // 隐式默认构造函数
 
struct T2
{
    int mem1;
    std::string mem2;
    T2(const T2&) {} // 用户提供的复制构造函数
};                   // 无默认构造函数
 
struct T3
{
    int mem1;
    std::string mem2;
    T3() {} // 用户提供的默认构造函数
};
 
std::string s{}; // 类 => 默认初始化，值为 ""
 
int main()
{
    int n{};                // 标量 => 零初始化，值为 0
    assert(n == 0);
    double f = double();    // 标量 => 零初始化，值为 0.0
    assert(f == 0.0);
    int* a = new int[10](); // 数组 => 每个元素的值初始化
    assert(a[9] == 0);      //          每个元素的值为 0
    T1 t1{};                // 有隐式默认构造函数的类 =>
    assert(t1.mem1 == 0);   //     t1.mem1 被零初始化，值为 0
    assert(t1.mem2 == "");  //     t1.mem2 被默认初始化，值为 ""
//  T2 t2{};                // 错误：类无默认构造函数
    T3 t3{};                // 有用户提供默认构造函数的类 =>
    std::cout << t3.mem1;   //     t3.mem1 被默认初始化为不确定值
    assert(t3.mem2 == "");  //     t3.mem2 被默认初始化，值为 ""
    std::vector<int> v(3);  // 值初始化每个元素
    assert(v[2] == 0);      // 每个元素的值为 0
    std::cout << '\n';
    delete[] a;
}

42

缺陷报告

下列更改行为的缺陷报告追溯地应用于以前出版的 C++ 标准。

缺陷报告 应用于 出版时的行为 正确行为
CWG 178 C++98 不存在值初始化；空初始化器会调用默认初始化（即使 new T() 也会进行零初始化） 空初始化器会调用值初始化
CWG 543 C++98 值初始化没有用户提供的构造函数的类对象与值初始化每个子对象
（对有用户提供的默认构造函数的成员不需要进行零初始化）等价		 先零初始化整个对象，
再调用默认构造函数
CWG 1301 C++11 值初始化有弃置的默认构造函数的结构体会导致零初始化 它们会被默认初始化
CWG 1368 C++98 不会在有任何用户提供的构造函数时进行零初始化 仅限默认构造函数
CWG 1502 C++11 值初始化没有用户提供的默认构造函数的联合体只会
零初始化整个对象，而不会处理默认成员初始化器		 在零初始化后
进行默认初始化
CWG 1507 C++98 值初始化没有用户提供的构造函数的类对象时不会

CWG 2820 C++98 默认初始化后的零初始化要求有非平凡构造函数 不要求有非平凡构造函数
CWG 2859 C++98 值初始化类对象在默认初始化实际不会选择用户提供的构造函数时也可能涉及零初始化 此时不会进行零初始化
