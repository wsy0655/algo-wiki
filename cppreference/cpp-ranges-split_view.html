1) split_view 接收一个 view 与一个分隔符，并按分隔符切割 view 为子范围。
2) 范围适配器对象 (RangeAdaptorObject) 。对于任何合适的子表达式 e 和 p，表达式 views::split(e, p) 表达式等价于 split_view(e, p)。

当底层 view V 实现概念 forward_range 和 common_range 时，split_view 实现对应概念。

内层范围（ranges::range_reference_t<split_view>）是一个实现了 common_range 的 ranges::subrange<ranges::iterator_t<V>>，并且在 ranges::iterator_t<V> 实现了 std::sized_sentinel_for<ranges::iterator_t<V>> 时实现 sized_range，并且在 V 实现了 contiguous_range，random_access_range，bidirectional_range 和 forward_range 时实现对应概念。

和 lazy_split_view 不同，split_view 可以保持子范围的连续性，使其适用于字符串分割。

注解

在 P2210R2 之前，split_view 在切分时使用了一种惰性 ﻿机制，这使得其无法保持底层视图的双向、随机访问和连续性质，也无法使内部范围的迭代器类型与底层视图相同。因此，它被 P2210R2 重新设计，而惰性机制被移动到了 lazy_split_view。

分隔符 pattern 通常不应当是普通字符串字面量，因为它会把空终止符也当做分隔符的必要部分；建议代之以使用 std::string_view 字面量。

示例

#include <iomanip>
#include <iostream>
#include <ranges>
#include <string_view>
 
int main()
{
    using std::operator""sv;
    constexpr auto words{"Hello^_^C++^_^20^_^!"sv};
    constexpr auto delim{"^_^"sv};
 
    for (const auto word : std::views::split(words, delim))
        // 使用 string_view 的 C++23 范围构造函数：
        std::cout << std::quoted(std::string_view(word)) << ' ';
    std::cout << '\n';
}

输出：

"Hello" "C++" "20" "!"
