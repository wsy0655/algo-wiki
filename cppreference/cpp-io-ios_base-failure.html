std::ios_base::failure

< cpp‎ | io‎ | ios base
在标头 <ios> 定义
class failure;

类 std::ios_base::failure 定义输入/输出库中的函数在失败时抛出的异常对象。

std::ios_base::failure 可以定义为 std::ios_base 的成员类，或拥有等价功能的另一个类的同义词（typedef）。

1,2) 以 message 作为解释字符串构造异常对象，能在之后用 what() 取得解释字符串。用 ec 鉴别特定的失败原因。(C++11 起)

注解

因为复制 std::ios_base::failure 不能抛出异常，通常将此消息在内部存储为分离分配的引用计数字符串。这也是构造函数不接收 std::string&& 形参的理由：无论如何它都必须复制内容。

注解

在解决 LWG 问题 331 前，std::ios_base::failure 有一个不带 throw() 的析构函数声明，但 std::exception::~exception() 的声明却带有 throw()[1]。这就意味着 std::ios_base::failure::~failure() 的异常声明更弱。解决方案是移除该声明，以保持不抛出的异常声明。

LWG 问题 363 针对的是同样的缺陷。它的解决方案是给 std::ios_base::failure::~failure() 的声明加上 throw()。该方案因为与前一个方案有冲突而没有被采纳。

  1. ↑ 这种不抛出异常声明现在从标准库全局应用，所以标准库类的析构函数的声明不会带有 throw() 或 noexcept。

示例

#include <fstream>
#include <iostream>
 
int main()
{
    std::ifstream f("不存在");
 
    try
    {
        f.exceptions(f.failbit);
    }
    catch (const std::ios_base::failure& e)
    {
        std::cout << "捕获了 ios_base::failure。\n"
                  << "解释字符串：" << e.what() << '\n'
                  << "错误码：" << e.code() << '\n';
    }
}

可能的输出：

捕获了 ios_base::failure。
解释字符串：ios_base::clear: 未指定的 iostream_category 错误
错误码：iostream:1
