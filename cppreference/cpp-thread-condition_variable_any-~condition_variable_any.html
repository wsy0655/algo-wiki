std::condition_variable_any::~condition_variable_any

< cpp‎ | thread‎ | condition variable any

~condition_variable_any();
 (C++11 起)

销毁 std::condition_variable_any 类型的对象。

注解

仅在已通知了所有线程后，调用析构函数才是安全的。不要求它们已退出其对应的等待函数：一些线程仍然可等待以重获得关联锁，或等待重获得后被调度运行。

一旦析构函数开始，程序员就必须确保不再有线程试图在 *this 上等待，特别要注意等待线程在循环中调用等待函数，或者使用接收谓词的等待函数重载的情况。
