1) 范围适配器，表示底层 view 的视图，也是 constant_range。as_const_view 总是具有只读元素（如果不为空）。
2) 范围适配器对象 (RangeAdaptorObject) 。令 e 是子表达式，令 T 是 decltype((e))，并令 U 是 std::remove_cvref_t<T>。那么表达式 views::as_const(e) 就表达式等价于：
  • views::all(e)，当它是良好定义的表达式且 views::all_t<T> 实现 constant_range，
  • 否则是 std::span<X, Extent>(e)，对于某个类型 X 和某个尺度 Extent，当 U 代表 std::span<const X, Extent>(e)；
  • 否则是 ranges::ref_view(static_cast<const X&>(e.base())，当对于某个类型 X，U 代表 ranges::ref_view<X>，且 const X 实现 constant_range；
  • 否则是 ranges::ref_view(static_cast<const U&>(e))，当 e 是左值，const U 实现 constant_range 且 U 不实现 view；
  • 否则是 as_const_view{e}。

as_const_view 始终实现 constant_range，并且当底层视图 V 实现 contiguous_range、random_access_range、bidirectional_range、forward_range、borrowed_range、common_range 和 sized_range 时，它也实现相应概念。

示例

#include <cassert>
#include <ranges>
 
int main()
{
    int x[]{1, 2, 3, 4, 5};
 
    auto v1 = x | std::views::drop(2);
    assert(v1.back() == 5);
    v1[0]++; // OK，可以修改非 const 元素
 
    auto v2 = x | std::views::drop(2) | std::views::as_const;
    assert(v2.back() == 5);
    // v2[0]++; // 编译错误，无法修改只读元素
}
