std::shared_lock<Mutex>::try_lock_for

< cpp‎ | thread‎ | shared lock
template< class Rep, class Period >
bool try_lock_for( const std::chrono::duration<Rep,Period>& timeout_duration );
 (C++14 起)

尝试以共享模式锁定关联互斥体。阻塞直至经过 timeout_duration 或获得锁这两个事件的先达成者为止。成功获得锁时返回 true，否则返回 false。相当于调用 mutex()->try_lock_shared_for(timeout_duration)。

由于调度或资源争夺延迟，此函数可能阻塞长于 timeout_duration。

标准建议使用稳定时钟。若实现用系统时钟代替，则等待时间亦可能对时钟调节敏感。

若无关联互斥体或互斥体已被锁定则抛出 std::system_error。

若 Mutex 不符合可共享定时锁定 (SharedTimedLockable) 要求则行为未定义。

参数

timeout_duration - 要阻塞的最大时长

返回值

若已成功获得互斥体的所有权则为 true，否则为 false。

异常

  • mutex()->try_lock_shared_for(timeout_duration) 所抛的任何异常。
  • 若无关联互斥体，则抛出以 std::errc::operation_not_permitted 为错误码的 std::system_error。
  • 若互斥体已被锁定，则抛出以 std::errc::resource_deadlock_would_occur 为错误码的 std::system_error。
