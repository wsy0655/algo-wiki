std::overflow_error

< cpp‎ | error
在标头 <stdexcept> 定义
class overflow_error;

定义作为异常抛出的对象类型。它能用于报告算术上溢错误（即计算结果对目标类型过大的情形）。

会抛出此异常的标准库组件只有 std::bitset::to_ulong。

 (C++11 前)

会抛出此异常的标准库组件只有 std::bitset::to_ulong 和 std::bitset::to_ullong。

 (C++11 起)

标准库组件的数学函数不会抛出此异常（数学函数按 math_errhandling 指定的方式报告上溢错误）。然而第三方库会使用它。例如，boost.math 在启用 boost::math::policies::throw_on_error（默认设置）时会抛出 std::overflow_error。

std::overflow_error::overflow_error

overflow_error( const std::string& what_arg );
 (1)
overflow_error( const char* what_arg );

 (3) (C++11 起为 noexcept)
1) 构造以 what_arg 作为解释字符串的异常对象。构造后 std::strcmp(what(), what_arg.c_str()) == 0。
2) 构造以 what_arg 作为解释字符串的异常对象。构造后 std::strcmp(what(), what_arg) == 0。
3) 复制构造函数。如果 *this 与 other 的动态类型都是 std::overflow_error，那么 std::strcmp(what(), other.what()) == 0。复制构造函数不能抛出异常。

参数

what_arg - 解释字符串
other - 要复制的另一异常对象

异常

1,2) 可能抛出 std::bad_alloc。

注解

因为不容许 std::overflow_error 的复制抛出异常，通常将此消息在内部存储为分离分配的引用计数字符串。这也是构造函数不接收 std::string&& 参数的理由：无论如何它必须复制内容。

在解决 LWG 问题 254 之前，非复制的构造函数只接受 std::string。这导致因需要构造 std::string 对象而不得不进行动态内存分配。

在解决 LWG 问题 471 之后，派生的标准异常类必须有一个公开可访问的复制构造函数。它可以隐式定义，只要分别在原对象和复制对象上通过 what() 获得的两个解释字符串相同即可。

示例

#include <iostream>
#include <limits>
#include <stdexcept>
#include <utility>
 
template<typename T, int N>
    requires (N > 0) /*...*/
class Stack
{
    int top_{-1};
    T data_[N];
 
public:
    [[nodiscard]] bool empty() const { return top_ == -1; }
 
    void push(T x)
    {
        if (top_ == N - 1)
            throw std::overflow_error("栈上溢出!");
        data_[++top_] = std::move(x);
    }
 
    void pop()
    {
        if (empty())
            throw std::underflow_error("栈下溢出!");
        --top_;
    }
 
    T const& top() const
    {
        if (empty())
            throw std::overflow_error("栈为空!");
        return data_[top_];
    }
};
 
int main()
{
    Stack<int, 4> st;
 
    try
    {
        [[maybe_unused]] auto x = st.top();
    }
    catch (std::overflow_error const& ex)
    {
        std::cout << "1) 异常: " << ex.what() << '\n';
    }
 
    st.push(1337);
    while (!st.empty())
        st.pop();
 
    try
    {
        st.pop();
    }
    catch (std::underflow_error const& ex)
    {
        std::cout << "2) 异常: " << ex.what() << '\n';
    }
 
    try
    {
        for (int i{}; i != 13; ++i)
            st.push(i);
    }
    catch (std::overflow_error const& ex)
    {
        std::cout << "3) 异常: " << ex.what() << '\n';
    }
}

输出：

1) 异常: 栈为空!
2) 异常: 栈下溢出!
3) 异常: 栈上溢出!

缺陷报告

下列更改行为的缺陷报告追溯地应用于以前出版的 C++ 标准。

缺陷报告 应用于 出版时的行为 正确行为
LWG 254 C++98 缺失了接受 const char* 的构造函数 已补充
LWG 471 C++98 std::overflow_error 的复制的解释字符串由实现定义 它们与原 std::overflow_error 对象的解释字符串相同
