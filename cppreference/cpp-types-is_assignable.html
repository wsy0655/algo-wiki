std::is_assignable, std::is_trivially_assignable, std::is_nothrow_assignable

< cpp‎ | types
在标头 <type_traits> 定义
template< class T, class U >
struct is_assignable;
 (1) (C++11 起)
template< class T, class U >
struct is_trivially_assignable;
 (2) (C++11 起)
template< class T, class U >
struct is_nothrow_assignable;
 (3) (C++11 起)
1) 如果表达式 std::declval<T>() = std::declval<U>() 在不求值语境中良构，则提供的成员常量 value 等于 true。否则，value 等于 false。如同从不关联到任一方的语境进行访问检查。
2) 同 (1)，但赋值表达式的求值将不调用任何非平凡的操作。为此检查的目的，对 std::declval 的调用被认为是平凡，且不是 std::declval 的 ODR 式使用。
3) 同 (1)，但赋值表达式的求值将不调用任何不是 noexcept 的操作。

如果 T 或 U 不是完整类型，（可有 cv 限定的）void 或未知边界数组，那么行为未定义。

如果上述模板的实例化直接或间接地依赖于不完整类型，并且假设在该类型完整的情况下实例化就会产生不同的结果，那么行为未定义。

如果程序添加了此页面上描述的任何模板的特化，那么行为未定义。

注解

此特征不检查赋值表达式的立即语境之外的任何事物：如果 T 或 U 的使用会触发模板特化，以及生成隐式定义的成员函数等等，而它们带有错误，则实际赋值可能无法编译，即使 std::is_assignable<T,U>::value 通过编译并求值为 true 也是如此。

示例

#include <iostream>
#include <string>
#include <type_traits>
 
struct Ex1 { int n; };
 
int main()
{
    std::cout << std::boolalpha
              << "int 可以从 int 赋值? "
              << std::is_assignable<int, int>::value << '\n' // 1 = 1; 无法编译
              << "int& 可以从 int 赋值? "
              << std::is_assignable<int&, int>::value << '\n' // int a; a = 1; 有效
              << "int 可以从 double 赋值? "
              << std::is_assignable<int, double>::value << '\n'
              << "int& 可以从 double 无抛出赋值? "
              << std::is_nothrow_assignable<int&, double>::value << '\n'
              << "string 可以从 double 赋值? "
              << std::is_assignable<std::string, double>::value << '\n'
              << "Ex1& 可以从 const Ex1& 平凡赋值? "
              << std::is_trivially_assignable<Ex1&, const Ex1&>::value << '\n';
}

输出：

int 可以从 int 赋值? false
int& 可以从 int 赋值? true
int 可以从 double 赋值? false
int& 可以从 double 无抛出赋值? true
string 可以从 double 赋值? true
Ex1& 可以从 const Ex1& 平凡赋值? true
