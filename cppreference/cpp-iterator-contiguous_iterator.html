std::contiguous_iterator

< cpp‎ | iterator
在标头 <iterator> 定义
template< class I >

    concept contiguous_iterator =
        std::random_access_iterator<I> &&
        std::derived_from</*ITER_CONCEPT*/<I>, std::contiguous_iterator_tag> &&
        std::is_lvalue_reference_v<std::iter_reference_t<I>> &&
        std::same_as<std::iter_value_t<I>,
                     std::remove_cvref_t<std::iter_reference_t<I>>> &&
        requires(const I& i) {
            { std::to_address(i) } ->
              std::same_as<std::add_pointer_t<std::iter_reference_t<I>>>;

        };
 (C++20 起)

contiguous_iterator 概念精化了 random_access_iterator，提供它代表的元素在内存中连续存储的保证。

给定具有某个实现了 contiguous_iterator 的类型的迭代器 i，哨位 s 和非负整数 n：

  • 对于任何范围 [i, s)，标准库函数都能将它替换成 [std::to_address(i), std::to_address(i + ranges::distance(i, s)))。
  • 对于任何范围 i + [​0​, n)，标准库函数都能将它替换成 std::to_address(i) + [​0​, std::to_address(i + n))。

这意味着程序不能依赖任何解引用或增减连续迭代器 i 的副作用，因为标准库函数可能会操作通过 std::to_address(i) 获取到的指针，而不是直接操作 i。

 (C++26 起)

迭代器概念确定

此概念的定义通过仅用于阐述的别名模板 /*ITER_CONCEPT*/ 说明。

为确定 /*ITER_CONCEPT*/<I>，令 ITER_TRAITS<I> 在特化 std::iterator_traits<I> 从主模板生成时代表 I，否则代表 std::iterator_traits<I>：

  • 如果 ITER_TRAITS<I>::iterator_concept 合法并指名类型，那么 /*ITER_CONCEPT*/<I> 代表该类型。
  • 否则，如果 ITER_TRAITS<I>::iterator_category 合法并指名类型，那么 /*ITER_CONCEPT*/<I> 代表该类型。
  • 否则，如果 std::iterator_traits<I> 从主模板生成，那么 /*ITER_CONCEPT*/<I> 指代 std::random_access_iterator_tag。
  • 否则 /*ITER_CONCEPT*/<I> 不代表类型并导致替换失败。

语义要求

设 a 和 b 为 I 类型的可解引用迭代器，c 为 I 类型的不可解引用迭代器，使得 b 从 a 可及且 c 从 b 可及。类型 I 只有在实现了 contiguous_iterator 所蕴含的所有概念并满足以下所有条件时才会实现 contiguous_iterator：

  • std::to_address(a) == std::addressof(*a)。
  • std::to_address(b) == std::to_address(a) + std::iter_difference_t<I>(b - a)。
  • std::to_address(c) == std::to_address(a) + std::iter_difference_t<I>(c - a)。
  • std::to_address(I{}) 具有良好定义。
  • ranges::iter_move(a) 与 std::move(*a) 具有相同的类型、值类别和效果。
  • 如果 ranges::iter_swap(a, b) 良构，那么它的效果等价于 ranges::swap(*a, *b)。

相等性保持

标准库概念的 requires 表达式中声明的表达式都要求保持相等性（除非另外说明）。

隐式表达式变种

使用了不修改某常量左值操作数的表达式的 requires 表达式，也会要求其隐式的表达式变种。

注解

每个指向完整对象类型的指针类型都实现了 contiguous_iterator。

在 C++17 中需要满足老式连续迭代器 (LegacyContiguousIterator) 的要求的标准库迭代器类型在 C++20 中同样需要实现 contiguous_iterator。
