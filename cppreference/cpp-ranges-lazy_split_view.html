1) lazy_split_view 接收一个 view 与一个分隔符，并按分隔符切割 view 为子范围。

支持两种主要场景：

  • 视图是 input_range，分隔符为单个元素（包装在 single_view 中）。
  • 视图是 forward_range，分隔符为元素的 view。
2) 范围适配器对象 (RangeAdaptorObject) 。表达式 views::lazy_split(e, f) 表达式等价于 lazy_split_view(e, f)。
3) 在 Pattern 满足 sized_range，Pattern::size() 是常量表达式且适合作为模板非类型实参，且 Pattern::size() 的值小于或等于 1 时，仅用于阐释的概念 /*tiny-range*/<Pattern> 得到满足。值得注意的是 empty_view 与 single_view 满足此概念。

当底层 view V 实现概念 forward_range 和 input_range 时，lazy_split_view 实现相应概念，并且当 V 一同实现 forward_range 与 common_range 时，lazy_split_view 实现 common_range。

当底层 view V 实现概念 forward_range 和 input_range 时，内层范围（ranges::range_reference_t<lazy_split_view>）实现相应的概念。它不实现 common_range，并且不能用于期待 bidirectional_range 或更高要求范围的算法。

与 split_view 不同，lazy_split_view 并不保持子范围的连续性。

推导指引

注解

名字 lazy_split_view 由 C++20 后的缺陷报告 P2210R2 引入。它拥有与改动前的旧 split_view 相同的惰性机制。

示例

#include <algorithm>
#include <iostream>
#include <ranges>
#include <string_view>
 
auto print = [](auto const& view)
{
    // `view` 是 std::views::lazy_split_view::__outer_iterator::value_type
 
    for (std::cout << "{ "; const auto element : view)
        std::cout << element << ' ';
    std::cout << "} ";
};
 
int main()
{
    constexpr static auto source = {0, 1, 0, 2, 3, 0, 4, 5, 6, 0, 7, 8, 9};
    constexpr int delimiter{0};
    constexpr std::ranges::lazy_split_view outer_view{source, delimiter};
    std::cout << "切割[" << std::ranges::distance(outer_view) << "]:  ";
    for (auto const& inner_view: outer_view)
        print(inner_view);
 
    constexpr std::string_view hello{"Hello C++ 20 !"};
    std::cout << "\n" "子字符串: ";
    std::ranges::for_each(hello | std::views::lazy_split(' '), print);
 
    constexpr std::string_view text{"Hello-+-C++-+-20-+-!"};
    constexpr std::string_view delim{"-+-"};
    std::cout << "\n" "子字符串: ";
    std::ranges::for_each(text | std::views::lazy_split(delim), print);
}

输出：

切割[5]:  { } { 1 } { 2 3 } { 4 5 6 } { 7 8 9 }
子字符串: { H e l l o } { C + + } { 2 0 } { ! }
子字符串: { H e l l o } { C + + } { 2 0 } { ! }
