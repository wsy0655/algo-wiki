1) basic_stacktrace 类模板表示整个栈踪迹或其给定部分的快照。它满足知分配器容器 (AllocatorAwareContainer) 、序列容器 (SequenceContainer) 及可逆容器 (ReversibleContainer) 的要求，但仅支持移动、复制、交换和 const 限定的顺序容器上的操作，并且比较函数的语义不同于对容器所要求的语义。
2) 使用默认 std::allocator 的 basic_stacktrace 的便利类型别名。
3) 使用多态分配器的 basic_stacktrace 的便利类型别名。

当前执行线程中的当前求值 \(\small{ {x}_{0} }\)x0 的调用序列，是求值的序列 \(\small{ ({x}_{0}, \dots, {x}_{n})}\)(x0, ..., xn)，它使得对于 \(\small{i \ge 0}\)i≥0，\(\small{ {x}_{i} }\)xi 在函数调用 \(\small{ {x}_{i+1} }\)xi+1 之内。

栈踪迹 ﻿是调用序列的近似表示，由栈踪迹条目组成。

每个栈踪迹条目 ﻿表示栈踪迹中的一项求值。它由 C++ 标准库中的 std::stacktrace_entry 表示。

模板形参

Allocator - 用于获取/释放内存及构造/销毁该内存中的元素的分配器。此类型必须符合分配器 (Allocator) 的要求。若 Allocator::value_type 不是 std::stacktrace_entry 则程序非良构。

注解

提供了对自定义分配器的支持，以便在热路径上或嵌入式环境中使用 basic_stacktrace。用户可以在栈上或其他适合的位置分配 stacktrace_entry 对象。

std::basic_stacktrace 所拥有的 std::stacktrace_entry 对象序列是不可变的，而且要么为空，要么表示整个栈踪迹的一个连续区间。

当 std::basic_stacktrace 不可用时可以代之以使用 boost::stacktrace::basic_stacktrace（于 Boost.Stacktrace 中可用）。

#include <iostream>
#include <stacktrace>
 
int nested_func(int c)
{
    std::cout << std::stacktrace::current() << '\n';
    return c + 1;
}
 
int func(int b)
{
    return nested_func(b + 1);
}
 
int main()
{
    std::cout << func(777);
}
