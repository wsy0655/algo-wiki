1-4) 隐式创建类型为 T、地址为 p 的完整对象以及内嵌于它的对象。以和调用 std::bit_cast<U>(E) 相同的方式确定每个被创建的可平凡复制 (TriviallyCopyable) 类型 U 的对象 obj 的值，但不实际访问其存储，其中 E 是指代 obj 的 U 类型的左值。否则，未指定这种被创建的对象的值。
  • T 必须为隐式生存期类型 (ImplicitLifetimeType) 且必须为完整类型，否则程序非良构。
  • 若符合以下条件则行为未定义：
  • [p, (char*)p + sizeof(T)) 不指代作为可由 p 抵达的存储区域的子集的已分配的存储区域，或
  • 该区域未对 T 恰当地对齐。
  • 注意未指定值可以为不确定值。
5-8) 隐式创建元素类型为 T、长度为 n 的数组。确切地说，若 n > 0 为 true 则等价于 std::start_lifetime_as<U>(p)，其中 U 为“n 个 T 的数组”。否则函数无效果。
  • T 必须为完整类型，否则程序非良构。
  • 若符合以下条件则行为未定义：
  • 非空的 p 未对 T 的数组恰当地对齐，或
  • n <= std::size_t(-1) / sizeof(T) 为 false，或
  • n > 0 且 [(char*)p, (char*)p + (n * sizeof(T))) 不指代作为可由 p 抵达的存储区域的子集的已分配的存储区域。

参数

p - 组成对象的区域的地址
n - 要创建的数组的元素数

返回值

1-4) 指向如上所述的完整对象的指针。
5-8) 指向被创建的数组的首元素的指针，若存在该数组；否则返回比较等于 p 的指针。

注解

new (void_ptr) unsigned char[size] 或 new (void_ptr) std::byte[size] 如 std::start_lifetime_as 的无类型版本一样起作用，但它不保持对象表示。

std::start_lifetime_as 处理非数组类型以及已知边界数组，而 std::start_lifetime_as_array 处理未知边界数组。

示例

#include <complex>
#include <iostream>
#include <memory>
 
int main()
{
    alignas(std::complex<float>) unsigned char network_data[sizeof(std::complex<float>)]
    {
        0xcd, 0xcc, 0xcc, 0x3d, 0xcd, 0xcc, 0x4c, 0x3e
    };
 
//  auto d = *reinterpret_cast<std::complex<float>*>(network_data);
//  std::cout << d << '\n'; // UB：network_data 不指向一个 complex<float>
 
//  auto d1 = *std::launder(reinterpret_cast<std::complex<float>*>(network_data));
//  std::cout << d1 << '\n'; // UB：隐式创建的对象具有动态存储期并具有不确定的初值，
//                              即使为其提供存储的数组具有确定的字节内容也是如此。
//                              另见 CWG2721。
 
    auto d2 = *std::start_lifetime_as<std::complex<float>>(network_data);
    std::cout << d2 << '\n'; // OK
}

可能的输出：

(0.1,0.2)
