std::deque<T,Allocator>::resize

< cpp‎ | container‎ | deque

void resize( size_type count );
 (1)
void resize( size_type count, const value_type& value );
 (2)

重设容器大小以容纳 count 个元素，在 count == size() 时不做任何事。

如果当前大小大于 count，那么减小容器到它的前 count 个元素。

如果当前大小小于 count，那么：

1) 追加额外的默认插入的元素。
2) 追加额外的 value 的副本。

参数

count - 容器的大小
value - 用以初始化新元素的值
类型要求
-
为使用重载 (1)， T 必须满足可移动插入 (MoveInsertable) 和可默认插入 (DefaultInsertable) 。
-
为使用重载 (2)， T 必须满足可复制插入 (CopyInsertable) 。

复杂度

与当前大小和 count 间的差成线性。

注解

如果不想要重载 (1) 中的值初始化，例如元素是非类类型且不需要清零，那么可以通过提供定制的 Allocator::construct 避免。

示例

#include <deque>
#include <iostream>
 
void print(auto rem, const std::deque<int>& c)
{
    for (std::cout << rem; const int el : c)
        std::cout << el << ' ';
    std::cout << '\n';
}
 
int main()
{
    std::deque<int> c = {1, 2, 3};
    print("deque 持有：", c);
 
    c.resize(5);
    print("在增加大小到 5 之后：", c);
 
    c.resize(2);
    print("在减少大小到 2 之后：", c);
 
    c.resize(6, 4);
    print("在增加大小到 6 之后（初始化式 = 4）：", c);
}

输出：

deque 持有：1 2 3
在增加大小到 5 之后：1 2 3 0 0
在减少大小到 2 之后：1 2
在增加大小到 6 之后（初始化式 = 4）：1 2 4 4 4 4

缺陷报告

下列更改行为的缺陷报告追溯地应用于以前出版的 C++ 标准。

缺陷报告 应用于 出版时的行为 正确行为
LWG 679 C++98 resize() 按值传递 value 按 const 引用传递
LWG 1418 C++98 未指定 resize(size()) 的行为 已指定
LWG 2033 C++11 1. 元素通过 erase() 移除[1]
2. T 不需要可移动插入 (MoveInsertable)		 1. 通过 pop_back() 移除
2. 需要
LWG 2066 C++11 重载 (1) 没有重载 (2) 的异常安全保证 已添加
LWG 2160 C++11 LWG 2033 的解决方案要求元素通过 pop_back() 移除[2] 不指定移除元素的方式
  1. ↑ erase() 可以从 deque 的中间移除元素，因此值类型需要可移动赋值 (MoveAssignable) ，这样在移除部分之后的元素可以向前移动以填补空缺。然而 resize() 只能从 deque 的末尾移除元素，这样就不需要可移动赋值 (MoveAssignable) 了。
  2. ↑ 通过 pop_back() 移除元素意味着这些元素需要从后往前移除。
